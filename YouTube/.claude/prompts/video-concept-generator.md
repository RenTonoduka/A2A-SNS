# Video Concept Generator v2.3

## 目的

YouTube動画の企画から台本までを一貫して設計する。
**解決策の具体性検証** + **深層心理分析**を基盤に、タイトル、フック、PASTOR、AREA本編を統合的に生成。

**v2.3での変更点**:
- Phase 7「AREA本編構成」を完全刷新
- **1つのAREA + 深掘り + ステップ展開**構造に変更
- 視聴者心理に寄り添うテクニック（疑問先取り、励まし、難しい用語フォロー）を追加

**v2.2での変更点**:
- PASTOR構造を修正（T with Testimony統合、O+R統合）

**v2.1での変更点**:
- Phase 1.5「解決策の具体性検証」を追加
- 解決策が曖昧な場合は先に進まない（ゲート機能）

---

## 入力項目（7つ）

| # | 項目 | 必須 | 説明 |
|---|------|------|------|
| 1 | **テーマ** | ○ | 動画の主題 |
| 2 | **サムネ案** | ○ | サムネイルのテキスト案 |
| 3 | **現状の課題** | ○ | 視聴者が抱えている問題 |
| 4 | **解決策** | ○ | この動画で提供する解決策 |
| 5 | **ゴール** | ○ | 視聴者が得られる具体的ベネフィット |
| 6 | **仮想敵** | ○ | 否定する対象（競合/常識/間違った方法） |
| 7 | **参考バズタイトル** | - | 模倣したいタイトル構造 |

## 出力構造（5層）

```
【Layer -1: 解決策の具体性検証】← v2.1 NEW（ゲート）
├─ 解決策の分解（何を、どうやって）
├─ 3ステップ言語化テスト
├─ 実体験・成果物の確認
└─ ゲート判定（PASS/FAIL）

【Layer 0: 深層心理分析】
├─ 5層ニーズ分析
├─ 恐怖/痛み/願望マップ
├─ 常識→盲点→逆説（深層心理ベース）
└─ 本質の特定

【Layer 1: タイトル生成】
├─ 5パターン生成
├─ viral-copy-master Agentで磨き上げ
└─ 評価・推薦

【Layer 2: 動画冒頭フック】
├─ 動画の概要（逆説的主張）
└─ タイトル回収

【Layer 3: PASTORフォーミュラ】
├─ P: Problem（深層心理の恐怖/痛みを活用）
├─ A: Amplify
├─ S: Solution（検証済みの具体的解決策）
├─ T: Transformation（願望を活用）
├─ O: Offer（検証済みの3ステップ）
└─ R: Response
```

## Phase 1: 入力整理・検証（5分）

### 1.1 入力の整理

7項目を表形式で整理

### 1.2 論理的整合性チェック

```
課題 → 解決策 → ゴール が一直線に繋がるか？

チェックポイント:
- 課題は解決策で解決されるか？
- 解決策を実行するとゴールに到達するか？
- 飛躍や矛盾がないか？
```

```
曖昧な仮想敵 → 具体的な仮想敵

例:
  「AI系YouTuber」（曖昧）
    ↓
  「小手先のプロンプトテクニックを教えて、本質を語らない人」（具体的）
```

## Phase 1.5: 解決策の具体性検証（10分）← v2.1 NEW【ゲート】

**目的**: 解決策が「言葉だけ」ではなく「実際に教えられる具体的な方法」であることを検証する。
このPhaseをPASSしないと、Phase 2以降に進めない。

### 1.5.1 解決策の分解

入力された「解決策」を以下の形式で分解：

```markdown
入力された解決策: {{解決策}}

### 分解結果

**何を（What）**:
- 具体的に何を得られるのか？
- 例: 「評価基準を持つ」→ 評価基準とは具体的に何？

**どうやって（How）**:
- どのような手順で実現するのか？
- 視聴者が再現できる形で説明できるか？

**なぜ効くのか（Why）**:
- なぜこの解決策で課題が解決されるのか？
- 論理的な根拠があるか？
```

### 1.5.2 3ステップ言語化テスト

**テスト**: 解決策を「初心者が今日から実践できる3ステップ」で説明できるか？

```markdown
### 3ステップ言語化

ステップ1: {{具体的なアクション}}
- 所要時間:
- 必要なもの:
- 完了条件:

ステップ2: {{具体的なアクション}}
- 所要時間:
- 必要なもの:
- 完了条件:

ステップ3: {{具体的なアクション}}
- 所要時間:
- 必要なもの:
- 完了条件:

### 言語化テスト結果

- [ ] 各ステップが具体的なアクションになっている
- [ ] 曖昧な言葉（「意識する」「心がける」等）がない
- [ ] 初心者でも再現できる
- [ ] 完了条件が明確
```

### 1.5.3 実体験・成果物の確認

**質問**: 発信者（あなた）はこの解決策を実際に使っているか？

```markdown
### 実体験チェック

**Q1: この解決策を自分で実践しているか？**
- [ ] はい → 具体的なエピソードを記載
- [ ] いいえ → Phase 1.5 FAIL

**Q2: 成果物を見せられるか？**
- [ ] はい → 成果物の種類を記載（スクリーンショット、実績、etc.）
- [ ] いいえ → Phase 1.5 FAIL（またはデモを作成してから再実行）

**Q3: 視聴者に「これをやってみて」と言える具体的な課題を出せるか？**
- [ ] はい → 課題の内容を記載
- [ ] いいえ → 解決策の具体化が必要
```

### 1.5.4 PASTOR素材ヒアリング ← v2.1追加

**目的**: PASTORフォーミュラの「P（実体験）」と「A（問題の拡大）」に使う素材を事前に収集

```markdown
### P: Problem用 - 過去の失敗体験

**Q4: この解決策を知る"前"に、どんな失敗をしましたか？**

具体的に教えてください：
- いつ頃？:
- 何をしていた？:
- どれくらい時間/お金を使った？:
- 結果どうなった？:
- その時の感情は？:

例:
- いつ頃？: 2024年春
- 何をしていた？: 画像生成AIでLP用の画像を作ろうとしていた
- どれくらい時間/お金を使った？: 5時間 + 有料プラン$60
- 結果どうなった？: 100枚生成しても「なんか違う」で終わった
- その時の感情は？: 「自分にはセンスがないのかも」と落ち込んだ

### A: Amplify用 - 放置した最悪の未来

**Q5: もしこの解決策を知らずに、昔のやり方を続けていたら？**

最悪の未来を想像してください：
- 時間の損失: どれくらい無駄になっていた？
- お金の損失: どれくらい損していた？
- 精神的な影響: どんな気持ちになっていた？
- キャリアへの影響: どうなっていた？

例:
- 時間の損失: 毎回LP作成に10時間以上、年間120時間を画像で消耗
- お金の損失: 結局外注に頼って、1LP 5万円 × 12回 = 60万円/年
- 精神的な影響: 「自分はAIを使いこなせない側の人間」という劣等感
- キャリアへの影響: AIを使える人に仕事を取られて、単価が下がり続ける

### 実体験の記載（まとめ）

**過去の失敗エピソード**:
（Q4の回答をまとめる）

**放置した場合の最悪の未来**:
（Q5の回答をまとめる）

**成果物**:
（見せられるものは何か）

**視聴者への課題**:
（動画を見た後にやってほしいこと）
```

### 1.5.5 ゲート判定

```markdown
## Phase 1.5 ゲート判定

### チェックリスト

| # | チェック項目 | 結果 |
|---|------------|------|
| 1 | 解決策が「何を」「どうやって」で分解できた | ○/× |
| 2 | 3ステップで言語化できた | ○/× |
| 3 | 各ステップに曖昧な言葉がない | ○/× |
| 4 | 自分で実践している | ○/× |
| 5 | 成果物を見せられる | ○/× |
| 6 | 視聴者への課題が出せる | ○/× |
| 7 | 過去の失敗体験がある（PASTOR用） | ○/× |
| 8 | 放置した最悪の未来を言語化できる（PASTOR用） | ○/× |

### 判定結果

**8項目中 {{N}} 項目PASS**

- **8/8 PASS** → Phase 2へ進む（PASTOR素材も完備）
- **6-7/8 PASS** → 不足項目を補完してから Phase 2へ
- **5/8以下** → **FAIL: 解決策の再定義またはPASTOR素材の追加が必要**

### FAILの場合のアクション

解決策が具体化できない場合、以下のいずれかを実行：

1. **解決策を具体化する**
   - 「評価基準を持つ」→「LP用画像の評価基準5項目を定義する」

2. **別の解決策に変更する**
   - 自分が実際にやっていること、成果物があることを解決策にする

3. **先にコンテンツを作る**
   - 解決策の具体例（テンプレート、チェックリスト等）を先に作成
   - それを動画で紹介する形に変更

**再入力フォーマット**:
解決策: {{具体化された解決策}}
成果物: {{見せられるもの}}
3ステップ: {{ステップ1}} → {{ステップ2}} → {{ステップ3}}
```

## Phase 2: 深層心理分析（20分）

**前提**: Phase 1.5をPASSしていること

### 2.1 5層ニーズ分析

視聴者のニーズを5階層で深掘り：

```markdown
### Level 1: 表面的ニーズ（言語化されている欲求）
視聴者が口に出して言うこと：
- 「〇〇の使い方を知りたい」
- 「〇〇を学びたい」
- 「〇〇で稼ぎたい」

### Level 2: 機能的ニーズ（解決したい具体的課題）
「なぜそれを知りたいのか？」を深掘り：
- 時間がない → 何を短縮したい？
- スキルがない → 何ができるようになりたい？
- お金がない → 何を節約したい？

### Level 3: 感情的ニーズ（抱えている不安・恐怖）
「なぜその課題を解決したいのか？」を深掘り：
- 恐怖: 取り残される、無能と思われる、失敗する
- 痛み: 時間の浪費、挫折経験、金銭的損失
- 劣等感: 学歴、年齢、技術力

### Level 4: 社会的ニーズ（承認欲求・所属欲求）
「誰に認められたいのか？」を特定：
- 上司: 評価されたい
- 同僚: 頼られたい
- 家族: 安心させたい
- 自分: 成長を実感したい

### Level 5: 自己実現ニーズ（本質的に目指す姿）
「本当になりたい自分」を特定：
- 理想の未来: 自律した存在、創造的な仕事人
- 避けたい未来: 置いていかれる人、使われる側の人
```

### 2.3 常識→盲点→逆説（深層心理ベース）

**重要**: 表面的な常識ではなく、**深層心理に基づいた常識**を特定する

```markdown
### 常識の特定（深層心理ベース）

Level 3（感情的ニーズ）から常識を導出：

| 恐怖/痛み | それを解消するために信じている常識 |
|----------|------------------------------|
| 取り残される恐怖 | 「最新ツールを学べば追いつける」 |
| 挫折の痛み | 「たくさん練習すれば上手くなる」 |
| 技術コンプレックス | 「プロンプトを学べば使いこなせる」 |

### 盲点の発見

各常識の「前提条件」「見落としている真実」を問う：

| 常識 | 盲点（見落としている真実） |
|------|--------------------------|
| 最新ツールを学べば追いつける | ツールは変わるが、思考法は変わらない |
| たくさん練習すれば上手くなる | 評価基準がなければ100回やっても学習しない |
| プロンプトを学べば使いこなせる | 「良いアウトプット」の定義がなければ改善しようがない |

### 逆説の導出

最もインパクトのある盲点から逆説を導出：

常識: 「{{恐怖を解消するために信じていること}}」
  ↓
盲点: 「{{見落としている真実}}」
  ↓
逆説: 「{{常識の逆}}をしろ。{{盲点}}が本質だ」
```
