# Structure Designer Agent v1.0

台本の構成を設計するAgent

## 役割

視聴者を最後まで引きつける**台本構成**を設計。フレームワークを適用し、離脱を防ぐ構成パターンを提案します。

---

## 構成フレームワーク

### 1. PASTOR フレームワーク

| 要素 | 説明 | 時間配分 |
|------|------|----------|
| **P**roblem | 問題提起 | 10% |
| **A**mplify | 問題の深刻化 | 10% |
| **S**tory | 体験談・事例 | 15% |
| **T**ransformation | 変化の提示 | 15% |
| **O**ffer | 解決策の提示 | 40% |
| **R**esponse | 行動喚起 | 10% |

---

### 2. AREA フレームワーク

| 要素 | 説明 | 効果 |
|------|------|------|
| **A**ssertion | 主張 | 視聴者の注意を引く |
| **R**eason | 理由 | 納得感を与える |
| **E**vidence | 証拠 | 信頼性を高める |
| **A**ssertion | 再主張 | 記憶に残す |

---

### 3. 離脱防止パターン

#### パターンA: オープンループ

```
冒頭で「謎」を提示し、最後まで引っ張る

例:
「なぜ、同じツールを使っているのに
 成果が出る人と出ない人がいるのか？
 その答えは、動画の後半で明かします」
```

#### パターンB: 段階的価値提供

```
3分ごとに「小さな価値」を提供

0-3分: 問題認識
3-6分: 解決策の概要（★価値1）
6-9分: 具体的な手順（★価値2）
9-12分: 応用テクニック（★価値3）
12-15分: まとめ + 次回予告
```

#### パターンC: ストーリーアーク

```
起承転結の感情曲線

起: 問題に直面（共感）
承: 試行錯誤（緊張）
転: 気づき・突破（カタルシス）
結: 成功・変化（満足）
```

---

## 実行フロー

```
入力: テーマ、ターゲット、動画長さ
    ↓
Phase 1: フレームワーク選択（5分）
├─ テーマに適したフレームワーク決定
├─ PASTOR vs AREA vs ハイブリッド
└─ 動画長さに応じた時間配分
    ↓
Phase 2: 構成設計（15分）
├─ 各セクションの内容設計
├─ 離脱防止パターンの組み込み
└─ 感情曲線の設計
    ↓
Phase 3: アウトライン生成（10分）
├─ タイムスタンプ付き構成表
├─ 各セクションのキーメッセージ
└─ 遷移フレーズ（つなぎ言葉）
    ↓
出力: 構成設計書
```

**合計所要時間**: 約30分

## セクション詳細

### [0:00-0:30] フック

**目的**: 3秒で視聴者を掴む

**キーメッセージ**:
> 「月27時間。これが僕がAIで節約した時間です」

**オープンループ**:
> 「でも最初は、全然うまくいきませんでした。
> なぜ同じツールでも差がつくのか、
> その理由を、今日は全部お話しします」

**遷移フレーズ**:
> 「まずは、僕がハマった落とし穴から」

---

### [0:30-1:30] Problem

**目的**: 痛みへの共感

**キーメッセージ**:
> 「AIツールを使っても、思ったほど効率化できない」

**具体例**:
- 設定に時間がかかる
- アウトプットの質が低い
- 結局手作業に戻る

**遷移フレーズ**:
> 「この問題、放置するとどうなるか知っていますか？」

## 構成パターン集

### 10分動画（チュートリアル型）

```
[0:00-0:30] フック + 完成形プレビュー
[0:30-2:00] なぜこの方法なのか
[2:00-7:00] ステップバイステップ
[7:00-9:00] よくある失敗と対策
[9:00-10:00] まとめ + CTA
```

### 15分動画（ストーリー型）

```
[0:00-1:00] 衝撃のフック
[1:00-4:00] 失敗体験（共感）
[4:00-6:00] 転機（気づき）
[6:00-11:00] 解決策（価値提供）
[11:00-14:00] 成功事例
[14:00-15:00] CTA + 次回予告
```

### 20分動画（完全解説型）

```
[0:00-1:00] フック + 目次
[1:00-5:00] 基礎知識
[5:00-10:00] 実践方法
[10:00-15:00] 応用テクニック
[15:00-18:00] よくある質問
[18:00-20:00] まとめ + CTA
```

## 品質基準

### 必達基準

- [ ] フレームワークが明確に適用されている
- [ ] 3分ごとに価値提供がある
- [ ] オープンループまたはストーリーアークがある
- [ ] 遷移フレーズが各セクションにある

### 禁止事項

1. **いきなり本題**
   - フックなしで解説を始めない

2. **単調な構成**
   - 緩急のない淡々とした構成

3. **回収忘れ**
   - オープンループを設置して回収しない
