# /script-improve コマンド

レビュー結果を基に台本を改善する

## 使用方法

```
/script-improve [台本ファイル] [レビューファイル]
/script-improve                      # 最新の台本とレビューを自動選択
```

## 実行手順

このコマンドは `script-improver-agent` を起動し、レビュー結果を基に台本を改善します。

---

### Step 1: 入力確認

**必要なファイル**:
1. 改善対象の台本（`scripts/drafts/` 内）
2. レビューレポート（`scripts/reviews/` 内）

自動選択の場合:
- 最新の台本ファイル
- 対応するレビューファイル

---

### Step 2: 改善優先度決定

**分析内容**:
1. レビュースコアの低い軸を特定
2. 改善インパクトの大きい順に並べ替え
3. 改善計画を策定

---

### Step 3: 各軸の改善

#### フック改善（スコア20未満の場合）

**テクニック**:
- 衝撃の事実から始める
- 問いかけから始める
- 数字から始める

#### 構成改善（スコア20未満の場合）

**テクニック**:
- PASTOR各フェーズの補完
- 感情曲線の最適化
- 接続部分の強化

#### 具体性改善（スコア15未満の場合）

**追加要素**:
- ツール名の明確化
- ステップ化
- 数値化
- 事例追加

#### 差別化改善（スコア10未満の場合）

**追加要素**:
- 独自の経験
- 独自の視点
- 独自のフレームワーク

#### CTA改善（スコア10未満の場合）

**追加位置**:
- 動画中盤（価値提供後）
- 動画終盤（まとめ前）
- 動画最後（行動喚起）

---

### Step 4: 最終調整

**確認事項**:
- 全体の整合性
- 冗長部分の削除
- 元の強みが維持されているか

---

### Step 5: 出力

**出力ファイル**:
```
scripts/drafts/YYYYMMDD_[テーマ]_v{N+1}.md
```

**変更ログ付き**:
- 主要な変更点
- Before/After 比較
- 改善理由

---

## 出力フォーマット例

```markdown
# 改善版台本

**改善日**: 2025-12-09
**元スコア**: 72/100
**バージョン**: v2

---

## 変更ログ

### 1. フック改善

**Before**:
> AIツールを使っていますか？今日は...

**After**:
> 月27時間。これ、先月僕がAIで節約した時間です。
> あなたはどうですか？

**理由**: 具体的な数字で興味を引く

### 2. 具体性追加

**追加データ**:
- 月27時間の時間節約
- 外注費10万円削減
- 作業効率3倍

### 3. CTA追加

**追加位置**: 動画中盤
> ここまでで「なるほど」と思った方、チャンネル登録しておくと次回も見逃しません

---

## 改善版台本

[完全な改善版台本]

---

## 次のステップ

この改善版を `/script-review` で再評価してください。
期待スコア: 85-95点
```

---

## 連携

- 入力: `/script-review` の出力
- 出力後: `/script-review` で再評価
- `/pipeline` から自動呼び出し可能

---

## 改善サイクル

```
v1台本 → レビュー(72点) → 改善 → v2台本
v2台本 → レビュー(88点) → 改善 → v3台本
v3台本 → レビュー(94点) → 採用!
```

**最大イテレーション**: 3回

---

## 注意事項

- レビューで評価された「強み」は変更しない
- 全ての変更に理由を記載
- 元台本より確実にスコア向上させる
