# /differentiate コマンド

競合動画との差別化戦略を設計

## 使用方法

```
/differentiate [テーマ]
```

## 実行手順

このコマンドは `differentiation-agent` を起動し、競合分析から独自の切り口を設計します。

---

### Step 1: 競合分析

**分析対象**:
- `/research` で抽出した優秀動画
- 同テーマの上位10動画

**分析項目**:
- タイトルのパターン
- 構成の共通点
- ターゲット層
- 視聴者コメントの不満

---

### Step 2: ギャップ発見

**5つの差別化軸で検討**:

1. **切り口**: 「使い方」vs「落とし穴」
2. **ターゲット**: 「エンジニア」vs「非エンジニア」
3. **深さ**: 「機能紹介」vs「業務への落とし込み」
4. **視点**: 「成功事例」vs「失敗→成功」
5. **フォーマット**: 「スライド」vs「実演」

---

### Step 3: 差別化戦略設計

**出力内容**:
- 「この動画だけの価値」定義
- 差別化ポイント3つ
- タイトル・台本・サムネイルへの反映案

---

## 出力例

```
## 差別化戦略

### この動画だけの価値

> 「非エンジニアの僕が、失敗から学んだAI活用の本当のコツ」

### 差別化ポイント3つ

1. **ターゲット**: 非エンジニア向け（競合はエンジニア中心）
2. **視点**: 失敗談から始める（競合は成功事例のみ）
3. **深さ**: 実際の業務フローへの落とし込み

### タイトル案

Before: 「〇〇ツールの使い方」
After: 「非エンジニアの僕が〇〇で月27時間節約できた理由」
```

---

## 連携

- 入力: `/research` の競合動画リスト
- 出力: `/video-concept` の差別化方針として使用
