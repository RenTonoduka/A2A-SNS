# Video Concept コマンド v2.1

YouTube動画の企画から台本冒頭までを一貫して設計します。

**v2.1での変更点**:
- Phase 1.5「解決策の具体性検証」を追加（ゲート機能）
- 解決策が曖昧な場合は先に進まない
- 「3ステップ言語化テスト」「実体験・成果物の確認」を必須化

**v2.0での変更点**:
- 深層心理分析（5層ニーズ、恐怖/痛み/願望マップ）を統合
- viral-copy-master、youtube-marketing-strategist Agentとの連携
- タイトル評価に「深層心理刺さり度」を追加

## 出力内容

```
【Layer -1: 解決策の具体性検証 + PASTOR素材】← v2.1 NEW（ゲート）
├─ 解決策の分解（何を/どうやって/なぜ）
├─ 3ステップ言語化テスト
├─ 実体験・成果物の確認
├─ PASTOR素材ヒアリング（過去の失敗/最悪の未来）
└─ ゲート判定（8項目中6項目以上でPASS）

【Layer 0: 深層心理分析】
├─ 5層ニーズ分析
├─ 恐怖/痛み/願望マップ（各5つ）
├─ 常識→盲点→逆説（深層心理ベース）
└─ 本質の特定 + JTBD分析

【Layer 1: タイトル】5パターン生成 + viral-copy-master磨き上げ + 評価 + 推薦
【Layer 2: フック】逆説的主張 + タイトル回収
【Layer 3: PASTOR】Problem → Amplify → Solution → **T with Testimony** → **O+R（統合）**
【Layer 4: AREA骨子】← v2.1 NEW
├─ 本編ポイント特定（1〜3点）
├─ A: Assertion（衝撃の結論）
├─ R: Reason（根拠を問いかけ）
└─ E: Example（具体例で気づき）
```

### Phase 1.5: 解決策の具体性検証 + PASTOR素材（15分）← v2.1 NEW【ゲート】

**目的**: 解決策が「言葉だけ」ではなく「実際に教えられる具体的な方法」であることを検証。
このPhaseをPASSしないと、Phase 2以降に進めない。

#### 1.5.1 解決策の分解

入力された「解決策」を以下の形式で分解：

**何を（What）**: 具体的に何を得られるのか？
**どうやって（How）**: どのような手順で実現するのか？視聴者が再現できる形で説明できるか？
**なぜ効くのか（Why）**: なぜこの解決策で課題が解決されるのか？

#### 1.5.2 3ステップ言語化テスト

**テスト**: 解決策を「初心者が今日から実践できる3ステップ」で説明できるか？

- ステップ1: {{具体的なアクション}} - 所要時間: / 必要なもの: / 完了条件:
- ステップ2: {{具体的なアクション}} - 所要時間: / 必要なもの: / 完了条件:
- ステップ3: {{具体的なアクション}} - 所要時間: / 必要なもの: / 完了条件:

**禁止ワード**（これらが入っていたらNG）:
- 〜的に、〜感を、〜力を、本質的、概念的、抽象的

#### 1.5.5 ゲート判定

| # | チェック項目 | 結果 |
|---|------------|------|

- **5/8以下 FAIL** → **解決策の再定義またはPASTOR素材の追加が必要。ユーザーに追加情報を求める。**
